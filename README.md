## Это интерактивное приложение, разработанное специально для оценки навыков тестировщиков (QA).
В нём кандидат проходит 6 практических кейсов, в каждом из которых нужно найти, локализовать и классифицировать баг в реальной бизнес-логике регистрации пользователя.

⸻

Установка и запуск

	1.	Убедитесь, что у вас установлены Node.js и npm.
	2.	Откройте терминал в корне проекта.
	3.	Установите зависимости: `npm install`
    4.	Запустите сервер: `npm start`
    5.	Откройте браузер и перейдите по адресу: http://localhost:3000
    

## Цель задания

Твоя задача — найти и локализовать серию багов в процессе регистрации пользователя.
Каждый кейс демонстрирует отдельную ошибку, встречающуюся в типичных веб-приложениях.

⚠️ Важно: не обращай внимания на ошибки валидации формы (например, пустые поля, неверные форматы и т.п.) — они считаются корректным поведением.
Тест фокусируется только на логике регистрации.

## Как проходить кейсы
Всего доступно 6 кейсов, на выполнение которых отводится 20 минут.

Для каждого кейса:

	1.	Ознакомься с поведением формы регистрации.
	2.	Найди причину, почему пользователь не может зарегистрироваться.
	3.	Кратко опиши:
	•	Описание бага — что именно работает некорректно (необходимо максимально локализовать баг);
	•	Модуль — Frontend или Backend.
	4.	Нажми кнопку “Next Case”, чтобы перейти к следующему сценарию.
	5.	После прохождения всех кейсов система покажет оценку точности твоих ответов.


⚠️ Важно:
Каждый кейс использует собственную версию Swagger-документации.
После перехода к следующему сценарию необходимо повторно открыть страницу API-документации (Swagger),
чтобы загрузилась актуальная версия эндпоинтов для текущего кейса.
Это гарантирует корректное отображение запросов и ответов, соответствующих выбранному сценарию.
